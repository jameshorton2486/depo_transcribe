## Composite Findings

**Strongest links.** The Deepgram leg of the pipeline is clean: word-level data flows from audio to JSON with full schema coverage (3,858 / 3,858 words have all seven expected fields), diarization separates 5 speakers, smart_format applies keyterm-driven capitalization (1,464 word/punctuated_word divergences), and the persisted raw JSON contains every artifact (utterances, raw_utterances, chunks, chunk_summaries, transcript) the downstream review panel would need. The yellow-highlight rendering path is also clean: 150 markers injected by `clean_format/low_confidence_markers.py` survived the round-trip and rendered as exactly 150 `<w:highlight w:val="yellow"/>` runs in the DOCX XML.

**Weakest links.** Three. (1) The `confirmed_spellings` dict is built and persisted by intake — 33 entries representing real human correction decisions — but never consumed in the active Start-Transcription path (see Active Path Audit Q2 and Q3). On this case Deepgram happened to produce the right forms anyway, so the gap is invisible; on a case where Deepgram missed a name, the dict is dead weight. (2) The cleanup pass emitted 58 `[SCOPIST: FLAG ...]` inline review notes directly into the DOCX body text rather than into a parallel review surface; these are visible to anyone reading the transcript. (3) The smoke run produced a Q:A paragraph ratio of 108:55 with 125 unclassified three-tab non-QA paragraphs, vs. the pre-smoke run's 106:102 with zero non-QA paragraphs from the same inputs — the structural-classification half of the cleanup pipeline behaves differently between runs in a way that is not yet explained.
